审查要点

- 计划符合性：确保代码完全按照附带的计划实现。
- 明显错误：查找代码中明显的错误或问题。
- 数据对齐问题：检查是否存在细微的数据格式不匹配，例如期望 snake_case 但得到 camelCase，或者期望接收一个对象但收到嵌套对象 {data:{}}。
- 过度工程化：查找是否有过度设计或文件过大需要重构的情况。
- 语法和风格一致性：查找任何奇怪的语法或风格，这些与代码库的其他部分不匹配（要求使用函数式编程风格）。

输出规范

- 将审查发现记录在 docs/features/N_[REVIEW.md](http://REVIEW.md) 文件中，除非用户指定了其他文件名。
